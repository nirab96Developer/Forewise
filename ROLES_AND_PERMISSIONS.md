## 5. ORDER_COORDINATOR — מתאם הזמנות (רכז הזמנות)

**תיאור:** תיאום בין פרויקטים לספקים. אחראי על שליחת הזמנות לספקים, מעקב אחר תגובות, ניהול סבב הוגן, ומעקב SLA.

### דפים בתפריט
| פריט | נתיב | הרשאה |
|------|-------|--------|
| ראשי (דשבורד) | `/` | `DASHBOARD.VIEW` |
| תיאום הזמנות | `/order-coordination` | `WORK_ORDERS.COORDINATE` |
| ספקים וציוד | `/settings/suppliers` | `SUPPLIERS.VIEW` |
| מפה | `/map` | `DASHBOARD.VIEW` |
| יומן אישי | `/my-journal` | `DASHBOARD.VIEW` |

### דשבורד
- KPI: ממתינות להפצה, בהפצה לספקים, ספק אישר, הושלמו
- Widgets: תור תיאום, הזמנות שפג תוקפן, סטטוס ספקים
- פעולות מהירות: תיאום הזמנות, ספקים

### מסך תיאום הזמנות (`/order-coordination`)
- רשימת כל הזמנות העבודה הממתינות לתיאום
- סינון לפי סטטוס: ממתינות, בהפצה, ספק אישר
- יכולת שליחת הזמנה לספק (יוצר portal token עם TTL 3 שעות)
- מעקב בזמן אמת אחר תגובת ספק
- העברה לספק הבא בסבב (כשספק דוחה)
- שליחה חוזרת לספק (הארכת זמן)
- ביטול הזמנות
- פעולות מרובות (bulk): מחיקה של הזמנות

### הרשאות (33 הרשאות)

**הזמנות עבודה:**
- צפייה, יצירה, עדכון
- אישור/דחייה, שיוך, הפצה לספקים
- תיאום (WORK_ORDERS.COORDINATE)
- הסלמה (WORK_ORDERS.ESCALATE)
- שליחה חוזרת (WORK_ORDERS.RESEND)

**ספקים:**
- צפייה, עדכון
- יצירת אילוצי ספקים

**פרויקטים:**
- צפייה בלבד

**ציוד:**
- צפייה (equipment, categories, types, scans)

**דיווחי עבודה:**
- צפייה בלבד

**דוחות:**
- צפייה, הרצה

**תמיכה:**
- יצירה, צפייה

### תהליך עבודה
1. נכנס לדשבורד — רואה KPI של הזמנות בתור תיאום
2. נכנס למסך תיאום הזמנות (`/order-coordination`)
3. רואה הזמנות עבודה מאושרות שממתינות לשליחה לספק
4. שולח הזמנה לספק ← מייל עם קישור לפורטל ספקים (תוקף 3 שעות)
5. מנטר תגובות:
   - ספק אישר → סטטוס `SUPPLIER_ACCEPTED_PENDING_COORDINATOR` → מתאם מאשר
   - ספק דחה → מעביר לספק הבא בסבב או מסלים
   - פג תוקף → שולח חוזר או מעביר לספק הבא
6. מעדכן אילוצי ספקים (זמינות, מגבלות גיאוגרפיות)
7. מנטר SLA — זמני תגובה, שיעורי דחייה

## 7. SUPPLIER — ספק

**תיאור:** גישת ספק חיצוני. ספקים לא מתחברים לאפליקציה הרגילה — הם מקבלים קישור ישיר לפורטל ספקים (ללא אימות רגיל).

### דפים
- **פורטל ספקים** (`/supplier-portal/:token`) — ללא אימות, גישה באמצעות token

> **הערה:** אם ספק מנסה להתחבר לאפליקציה הראשית, המערכת מנקה את הסשן ומחזירה אותו לדף ההתחברות.

### פורטל הספקים
- אין תפריט צד — דף נחיתה בודד
- מציג את פרטי הזמנת העבודה:
  - מספר הזמנה, כותרת, תיאור
  - סוג ציוד, שעות משוערות, תעריף, סכום כולל
  - שם פרויקט, מרחב, אזור
  - תאריכי עבודה
  - זמן שנותר לתגובה (TTL 3 שעות)

### פעולות בפורטל
| פעולה | תיאור |
|-------|-------|
| **אישור הזמנה** | ספק בוחר כלי (ציוד), מספר רישוי, הערות → סטטוס הופך ל-`SUPPLIER_ACCEPTED_PENDING_COORDINATOR` |
| **דחיית הזמנה** | ספק בוחר סיבת דחייה + הערות → ההזמנה עוברת לספק הבא בסבב (או נדחית אם בחירה כפויה) |
| **בדיקת סטטוס** | בדיקת תוקף הטוקן, זמן שנותר |

### הרשאות (15 הרשאות)
- `DASHBOARD.VIEW`
- `EQUIPMENT.VIEW`, `equipment.read`
- `INVOICES.CREATE`, `INVOICES.VIEW`, `invoices.read_own`
- `invoice_payments.read_own`
- `PROJECTS.VIEW`
- `WORKLOGS.CREATE`, `WORKLOGS.UPDATE`
- `worklogs.read_own`, `worklogs.submit`
- `WORK_ORDERS.VIEW`, `work_orders.read_own`
- `support_tickets.create`

### תהליך עבודה
1. מקבל מייל עם קישור לפורטל ספקים (token תקף 3 שעות)
2. נפתח דף הפורטל → רואה את פרטי הזמנת העבודה
3. בוחר: אישור (+ בחירת כלי) או דחייה (+ סיבה)
4. אם אישר → מתחיל עבודה בשטח
5. מדווח שעות עבודה (worklogs) דרך האפליקציה
6. ממתין לאישור הדיווח ע"י מנהל העבודה / מנהל אזור / מנהלת חשבונות
7. לאחר אישור → מקבל PDF אישור בדוא"ל
8. חשבונית נוצרת ע"י מנהלת החשבונות

## מחזור הזמנת עבודה (Work Order Lifecycle)

```
┌──────────┐     ┌──────────┐     ┌──────────────┐     ┌──────────────────┐
│  DRAFT   │ ──→ │ PENDING  │ ──→ │   APPROVED   │ ──→ │ APPROVED_AND_SENT│
│ (טיוטה)  │     │ (ממתין)  │     │  (מאושר)     │     │ (אושר ונשלח)     │
└──────────┘     └──────────┘     └──────────────┘     └──────────────────┘
                       │                                        │
                       │ (נדחה)                                 ▼
                       ▼                              ┌──────────────────┐
                 ┌──────────┐                         │  DISTRIBUTING    │
                 │ REJECTED │                         │ (בהפצה לספקים)   │
                 │ (נדחה)   │                         └──────────────────┘
                 └──────────┘                                  │
                                                               ▼
                                                    ┌──────────────────────────────┐
                                                    │   PENDING_SUPPLIER           │
                                                    │   (ממתין לספק)               │
                                                    └──────────────────────────────┘
                                                         │              │
                                              (ספק אישר) │              │ (ספק דחה / פג תוקף)
                                                         ▼              ▼
                                        ┌────────────────────────┐   ┌──────────┐
                                        │ SUPPLIER_ACCEPTED_     │   │ EXPIRED  │
                                        │ PENDING_COORDINATOR    │   │(פג תוקף) │
                                        │(ספק אישר—ממתין למתאם) │   └──────────┘
                                        └────────────────────────┘
                                                    │
                                                    ▼
                                            ┌──────────────┐
                                            │  ACCEPTED    │
                                            │  (אושר)      │
                                            └──────────────┘
                                                    │
                                                    ▼
                                            ┌──────────────┐
                                            │ IN_PROGRESS  │
                                            │  (בתהליך)    │
                                            └──────────────┘
                                                    │
                                                    ▼
                                            ┌──────────────┐
                                            │  COMPLETED   │
                                            │  (הושלם)     │
                                            └──────────────┘
```

| העברה לספק הבא | `work_orders.update` | ADMIN, ORDER_COORDINATOR |
